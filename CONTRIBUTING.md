The following is a set of guidelines for contributing to Self-Host-Link. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if it has already been reported.
* Use the **Bug Report** template.
* Include as much detail as possible: steps to reproduce, browser, OS, and screenshots.

### Suggesting Enhancements
* Check the existing issues for similar suggestions.
* Use the **Feature Request** template.
* Describe the goal and why this would be useful for others.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style (run `npm run lint`).
5. Issue that PR!

## Styleguide
* We use TypeScript for everything.
* We use Tailwind CSS for styling.
* Try to keep components small and focused.

## Local Development
1. `npm install`
2. `npm run dev`

Thanks!
